# Visual Direction Overview
## Six projects · one studio mind

Each project owns its palette, photography genre, and temperament.  
Codexify owns pacing, honesty, and typographic discipline.

---

## Spectrum

```
DARK / OBJECT     Atelier Noir
LIGHT / SPACE     Sora Residence
RHYTHM / IMAGE    Velora
COOL / TRUST      Nexa Capital
SOFT / CARE       Auren Clinic
WARM / MATERIAL   Forma Studio
```

Reading the exhibition top to bottom should feel like moving through different rooms of one museum — not six random Behance shots.

---

## Shared rules (all projects)

1. **No device mockup heroes** — type, material, product, architecture, or system graphics
2. **One accent family** per brand — no rainbow UI
3. **Motion serves meaning** — still is a valid choice (especially Nexa, Atelier)
4. **UI chrome stays quiet** — the brand is loud only through its own art direction
5. **Thumbnails = identity fragments** — letterform, texture, fragment of a room, dial crop — never a tiny screenshot of a full homepage

## Codexify site chrome vs. project skins

When case studies are built later:

- **Shell** (nav, footer, page transitions): Codexify system (Peyda, canvas, terracotta accent sparingly)
- **Article body art direction**: may adopt project ground/ink for immersive chapters — if so, transition in and out deliberately so the studio identity doesn’t dissolve

Recommendation for first build: keep Codexify shell constant; let project color live in media and selected full-bleed breaks — safer, clearer authorship.

## Quality bar

If two projects could swap thumbnails without anyone noticing, the identities failed.  
If six projects feel like six different agencies, Codexify failed.  
Hold both truths.
