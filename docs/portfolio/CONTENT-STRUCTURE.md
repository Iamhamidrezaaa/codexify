# Case Study — Content Structure
## Field map for every future project

Use this as the content checklist when writing or implementing a case study.  
Fill every field. Leave none as “TBD” in published work.

---

## Meta

| Field | Example |
|-------|---------|
| `slug` | `atelier-noir` |
| `name` | Atelier Noir |
| `industry` | Luxury watches |
| `year` | 2025 |
| `statement` | One sentence thesis |
| `order` | Exhibition index 01–06 |
| `keywords` | 5–8 search/editorial keywords |

---

## Hero

- name  
- industry  
- year  
- statement  

---

## Challenge

- problem_summary  
- why_redesign  
- constraints (optional, honest)  

---

## Discovery

- research_notes  
- positioning  
- audience  
- strategy_decision  

---

## Creative Direction

- mood (list)  
- typography_direction  
- composition  
- color_intent  
- references (described)  

---

## Design System

- grid  
- spacing  
- components (list)  
- interaction_principles  

---

## Selected Screens

| Breakpoint | Frames to show | Why these |
|------------|----------------|-----------|
| Desktop | 2–4 | Primary narrative |
| Tablet | 0–2 | Only if system shifts |
| Mobile | 1–3 | Hierarchy proof |

Caption each frame with the *decision*, not the page title alone.

---

## Interaction Highlights

For each:

- name  
- purpose (one line)  
- why_it_belongs_to_this_brand  

Max 3 per project. Often 1–2 is enough.

---

## Outcome

- business_impact (qualitative)  
- brand_perception  
- experience_quality  

No fabricated percentages.

---

## Asset direction (production later)

| Asset type | Guidance |
|------------|----------|
| Index visual | Per project visual direction doc |
| Hero media | Brand-led, no device frame |
| In-article frames | Cropped UI or art-directed stills |
| Avoid | Stock “team smiling”, generic laptop PNGs |

---

## Voice checklist before publish

- [ ] Any sentence could appear in an architecture journal  
- [ ] No banned marketing phrases  
- [ ] Challenge is real; outcome is honest  
- [ ] Interactions are justified  
- [ ] Project feels distinct; still feels like Codexify
